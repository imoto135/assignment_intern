# 技術課題

## 本課題について

本課題は、クライアント企業からPoC（技術検証）案件の相談を受けたという想定で取り組んでいただく技術課題です。  
単なるモデル実装ではなく、受託案件における初期PoCフェーズをどのように設計・推進できるかを評価します。

## 想定案件概要（クライアントからの相談内容）

```
当社では、電力用変圧器の安定運用および保全高度化を目的として、各種センサデータを活用したデータドリブンな運用改善に取り組んでいます。
特に変圧器のオイル温度は、設備劣化や事故リスクと密接に関係する重要な指標です。

これまで当社では、経験則に基づいて単純な閾値監視を中心に運用してきましたが、近年は以下のような課題感を持っています。

- 負荷変動や外部要因の影響により、温度変動が複雑化している  
- 事後対応型の保全から、予測に基づく予防保全へ移行したい  
- 機械学習・時系列予測技術の進展により、実運用で使える可能性が見えてきた  

そこで今回、公開ベンチマークであるETT（r Temperature）データセットを用い）Electricity）データセットを用いて、
「オイル温度を将来予測することで、実際の運用・保全判断にどの程度価値を出せそうか？」
を検証するPoCを依頼したいと考えています。
```

## 技術課題内容

あなたは、このPoCを担当するデータサイエンティスト／機械学習エンジニアです。
以下の観点で技術検証を進め、その内容をクライアントに報告してください。

1. EDAを通じて、データからどのような示唆が得られたか  
2. それを踏まえて、どのようなアプローチが妥当だと考えたか  
3. 実際にモデルを構築した結果、何が分かったか  
4. 工夫した点・設計上の意思決定は何か  
5. 現時点での限界や課題、および今後の改善方針は何か

## 使用データ

- ETT（Electricity Transformer Temperature）データセット
  - 本リポジトリの `data/` 内のデータを使用してください
  - 詳細は [zhouhaoyi/ETDataset](https://github.com/zhouhaoyi/ETDataset/blob/main/README.md) を参照してください
  - 追加データの利用は任意ですが、使用する場合は出典を明記してください
